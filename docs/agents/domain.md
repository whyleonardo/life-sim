# Domain Docs

## Context

Read `CONTEXT.md` at the repository root for the project's domain language, key concepts, and terminology.

## Architectural Decision Records

Read `docs/adr/` for past architectural decisions. Each ADR follows the format:

- `NNNN-kebab-case-title.md` — numbered, titled, with status (Proposed / Accepted / Deprecated / Superseded)

When making architectural decisions, create a new ADR in `docs/adr/` following this convention.
